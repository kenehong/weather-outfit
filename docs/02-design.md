# Weather Outfit — 디자인 컨셉

## 개요

온도 숫자를 모르는 아이들이 캐릭터의 옷차림만 보고 오늘 날씨를 직관적으로 이해할 수 있는 날씨 앱.
Gemini AI로 제작한 캐릭터 이미지를 활용하며, 날씨에 따라 배경과 캐릭터 의상이 바뀐다.

---

## 디자인 원칙

### 1. 글자 없이도 이해되는 UI (Glanceable)
숫자와 텍스트를 읽지 않아도 캐릭터 이미지 + 배경색만으로 날씨를 파악할 수 있어야 한다.
텍스트는 보조 수단이지 핵심 전달 채널이 아니다.

### 2. 한 화면, 한 가지 정보 (Single Focus)
화면에 표시되는 정보량을 최소화한다. 아이들은 복잡한 대시보드를 읽지 않는다.
핵심 정보(캐릭터 + 옷차림 조언)가 화면의 70% 이상을 차지해야 한다.

### 3. 큰 터치, 쉬운 조작 (Kid-Proof)
모든 인터랙션은 탭 한 번으로 완료. 스와이프, 롱프레스, 더블탭 없음.
터치 타겟 최소 48px. 화면 하단에 주요 액션 배치 (엄지 영역).

```
[앱 실행]
    |
    v

    |                           |
    |거부                       v
    v                    [메인 화면 표시]
[장소 검색 화면]          - 캐릭터 (날씨 맞는 옷)
                          - 배경 (날씨 맞는 색/분위기)
                          - 텍스트 ("자켓 입어!")
                          - 온도 (작게)
```

```
[메인 화면]
    |
    v

    |
    v

    |
    v

    |
    v

    |

    - 배경 변경

    |
    v
[뒤로가기] --> [현재 위치 날씨로 복귀]
```

## 와이어프레임 -- 컨셉 A: "캐릭터 풀스크린"

**컨셉**: 캐릭터가 화면 전체를 지배. UI 요소는 최소한으로 캐릭터 위에 오버레이.
카카오톡 이모티콘이나 라인프렌즈 같은 느낌. 앱이라기보다 그림책 한 페이지.

### 특징
- 캐릭터 이미지가 화면의 60-70% 차지
- UI 요소는 상단(위치+검색)과 하단(말풍선+온도)에만 존재
- 배경 전체가 날씨에 따라 바뀜 (그라디언트)
- 말풍선이 캐릭터가 말하는 것처럼 보임

### 장점
- 가장 직관적. 앱 열면 바로 이해됨
- 아이들이 캐릭터에 집중하기 쉬움
- 구현이 가장 단순

### 단점
- 정보 밀도가 낮음 (시간대별 날씨 등 확장 어려움)
- 캐릭터 이미지 품질에 앱 퀄리티가 크게 좌우됨
- 장소 비교가 한 번에 안 됨

## 와이어프레임 -- 컨셉 C: "장면 속 캐릭터"

**컨셉**: 캐릭터가 날씨 장면(scene) 안에 서 있는 일러스트 느낌.

- 캐릭터가 독립적이지 않고 "장면(scene)" 안에 배치됨
- 날씨마다 완전히 다른 장면 (비 장면, 눈 장면, 맑은 날 공원 등)
- 이미지 한 장이 날씨의 모든 것을 전달
- 하단에 최소한의 텍스트 정보만

### 장점
- 가장 감성적이고 몰입감 있음
- 장면 자체가 날씨를 설명해서 텍스트 의존도 최저
- 아이들이 "그림책" 보듯이 즐겁게 사용
- 날씨별 장면이 수집 요소가 될 수 있음 ("오늘은 어떤 장면일까?")

### 단점
- 이미지 제작 비용이 가장 높음 (날씨 x 장면 조합)
- Gemini로 일관된 장면 스타일 유지가 어려울 수 있음
- 캐릭터 단독 vs 장면 이미지 -- 이미지 관리 복잡도 증가

## 와이어프레임 -- 컨셉 D: "감정 날씨" (보너스)

**컨셉**: 캐릭터의 표정/감정이 날씨를 대변. 더우면 땀, 추우면 떨림, 비면 우울.
배경은 단색 그라디언트로 극도로 단순하게. 캐릭터 표정 + 한 줄 텍스트가 전부.

### 모바일 (375 x 812)

```
+----------------------------------+
|                                  |
|                                  |
|                                  |  <- 배경 전체 = 날씨 컬러
|                                  |      (파랑 그라디언트 = 추움)
|         +--------------+        |
|         |              |        |
|         |  캐릭터      |        |
|         |  (크게)      |        |
|         |              |        |
|         |  🥶 떨고있는 |        |
|         |    표정       |        |
|         |              |        |
|         +--------------+        |
|                                  |
|                                  |
|      "따뜻하게 입어!"            |  <- 큰 텍스트 (24px+)
|                                  |
|  시애틀 · 3°C                    |  <- 작은 서브
|                                  |
|  [다른 곳 보기]          [검색]  |  <- 하단 버튼 영역
+----------------------------------+
```

### 태블릿 / 데스크톱

모바일과 동일한 구조를 max-width: 480px로 센터 정렬. 이 컨셉은 의도적으로 모바일 전용 느낌을 유지.

```
+---------------------------------------------------+
|                                                     |
|           +-----------------------------+           |
|           |                             |           |
|           |       (날씨 컬러 배경)      |           |
|           |                             |           |
|           |      +-----------+          |           |
|           |      |  캐릭터   |          |           |
|           |      |  🥶 떨림  |          |           |
|           |      +-----------+          |           |
|           |                             |           |
|           |   "따뜻하게 입어!"          |           |
|           |   시애틀 · 3°C              |           |
|           |                             |           |
|           |   [다른 곳 보기]   [검색]   |           |
|           +-----------------------------+           |
|                                                     |
+---------------------------------------------------+
```

### 특징
- 가장 미니멀. 화면에 캐릭터 + 한 줄 텍스트 + 배경색 세 가지만
- 캐릭터의 표정/감정이 핵심 (옷 + 표정 조합)
- 배경은 순수 그라디언트 (이미지 없음)
- 검색은 하단 버튼으로

### 장점
- 구현이 가장 단순하고 로딩이 빠름
- 배경이 단색이라 캐릭터 이미지가 가장 돋보임
- 이미지 한 장에 옷+표정 모두 담기면 됨 (장면 배경 불필요)
- 감정 표현이 아이들에게 매우 직관적

### 단점
- 정보가 거의 없음 (시간대별 날씨 등 확장 어려움)
- 너무 단순해서 "앱"보다 "위젯" 느낌일 수 있음
- 매일 열어보는 습관을 만들 만한 콘텐츠가 부족

## UX 고려사항

### 아이들 사용 최적화

1. **터치 타겟**: 모든 인터랙티브 요소 최소 48x48px. 검색 버튼은 56x56px.
2. **텍스트 크기**: 메인 조언 텍스트 24px 이상. 아이들이 읽기 쉬운 큰 글씨.
3. **색상 대비**: 모든 텍스트 WCAG AA 이상. 배경 그라디언트 위 텍스트는 반투명 카드/말풍선 안에 배치하여 대비 확보.
4. **단순한 동선**: 앱 열기 → 바로 정보 확인. 추가 탭 없음. 검색만 유일한 서브 화면.
5. **실수 방지**: 뒤로가기, 새로고침 등 명확한 네비게이션. "돌아가기" 버튼은 항상 보이게.

### 위치 권한

| 상태 | 처리 |
|------|------|
| 첫 실행, 권한 미요청 | 친근한 설명 + 권한 요청 ("네가 어디 있는지 알려줘!") |
| 권한 허용 | 자동으로 현재 위치 날씨 표시 |
| 권한 거부 | 검색 화면으로 이동 (장소 직접 입력) |
| 위치 오류 | "위치를 못 찾겠어... 직접 알려줘!" + 검색 |

### 로딩 상태

- 캐릭터 이미지 자리에 실루엣 스켈레톤 (캐릭터 형태의 회색 플레이스홀더)
- 배경은 중립 그라디언트 (회색→흰색) 유지
- 텍스트 영역: 둥근 사각형 스켈레톤 2줄

### 에러 상태

- 네트워크 오류: 캐릭터가 혼란스러운 표정 + "날씨를 못 가져왔어... 다시 해볼까?" + [다시 시도] 버튼
- API 오류: 동일 처리
- 위치 오류: 검색으로 유도

### 빈 상태

- 검색 결과 없음: "그 장소를 모르겠어... 다시 찾아볼까?" + 검색 입력 포커스
- 저장된 장소 없음 (컨셉 B에서): "아직 저장한 곳이 없어!"

### 접근성

- 캐릭터 이미지에 의미 있는 alt 텍스트: "자켓을 입은 캐릭터, 오늘은 쌀쌀해요"
- 배경색만으로 날씨 구분하지 않음 -- 항상 텍스트 병행
- 키보드 탭 순서: 검색 버튼 → 위치명 → 메인 콘텐츠 → 하단 정보
- focus-visible 링: `outline: 3px solid oklch(65% 0.15 250); outline-offset: 2px;`
- `prefers-reduced-motion`: 배경 전환 애니메이션 비활성화, 즉시 전환

### 검색 화면 상세

```
+----------------------------------+
|  [← 뒤로]                       |
|                                  |
|  +----------------------------+  |
|  | 🔍 장소 이름 입력...       |  |  <- 자동 포커스
|  +----------------------------+  |
|                                  |
|  최근 검색                       |
|  +----------------------------+  |
|  | 📍 서울                    |  |
|  | 📍 뉴욕                    |  |
|  | 📍 도쿄                    |  |
|  +----------------------------+  |
|                                  |
|  (입력 시 자동완성 목록 표시)    |
|  +----------------------------+  |
|  | 시애틀, WA, US             |  |
|  | 시애틀, 영국               |  |
|  +----------------------------+  |
+----------------------------------+
```

- 검색 입력 시 300ms 디바운스 후 자동완성
- 최근 검색 최대 5개 (localStorage)
- 자동완성 항목 터치 타겟 48px+ (패딩 포함)
- 검색어 없을 때 "최근 검색" 표시
- 검색어 입력 시 자동완성으로 전환

### 모션 / 전환

- 날씨 변경 시 배경 그라디언트 전환: `transition: background 600ms ease`
- 캐릭터 이미지 전환: 페이드 (opacity 300ms)
- 검색 화면 진입: 슬라이드 업 (transform 300ms)
- `prefers-reduced-motion` 시 모든 전환 즉시 적용 (duration: 0ms)
