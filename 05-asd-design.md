# ASD 맞춤 디자인 — Weather Outfit (Smart Display)

> 자폐 스펙트럼 장애(ASD) 사용자(5세~청소년)를 위한 UX 적응 설계
> 대상 디바이스: Echo Show, Google Nest Hub (가로 모드)
> 작성일: 2026-03-19

---

## 🧩 ASD 맞춤 디자인

### 설계 원칙

1. **예측 가능성(Predictability)**: 모든 화면 전환, 상태 변화, 레이아웃이 일관되어야 한다. 예고 없는 변화는 불안을 유발한다.
2. **감각 조절(Sensory Regulation)**: 시각적 자극의 양을 사용자가 통제할 수 있어야 한다. 기본값은 "적은 자극"이다.
3. **구조화된 단계(Structured Steps)**: 정보를 한 번에 주지 않는다. "지금 날씨 → 뭘 입을까 → 밖에 나가자" 순서로 안내한다.
4. **나이 적응(Age Adaptation)**: 같은 앱이 5세와 15세를 모두 지원해야 한다. 복잡도 레벨로 해결한다.

---

### 감각 설정 시스템

보호자가 초기 설정에서 감각 프로필을 구성한다. 이후 사용자가 화면에서 빠르게 전환할 수 있는 토글도 제공한다.

#### 감각 프로필 3단계

| 프로필 | 이름 | 대상 | 설명 |
|--------|------|------|------|
| **Standard** | 기본 모드 | 감각 민감도 낮은 사용자 | 02-design.md의 기본 디자인 그대로. 날씨 파티클, 배경 전환 애니메이션 모두 활성 |
| **Calm** | 차분한 모드 | 감각 민감도 중간 | 파티클 제거, 애니메이션 속도 50% 감소, 채도 30% 감소, 소리 없음 |
| **Still** | 정적 모드 | 감각 민감도 높은 사용자 | 모든 애니메이션 제거, 단색 배경, 정적 이미지만 표시 |

#### 각 프로필별 구체적 차이

**배경(Scene)**

| 요소 | Standard | Calm | Still |
|------|----------|------|-------|
| 비 파티클 | 빗방울 애니메이션 | 없음 (정적 회색 줄무늬 오버레이) | 없음 (단색 배경) |
| 눈 파티클 | 눈송이 떨어짐 | 없음 (정적 흰 점 오버레이) | 없음 (단색 배경) |
| 배경 전환 | 600ms 크로스페이드 | 1200ms 크로스페이드 | 즉시 전환 (0ms) |
| 태양 펄스 | 밝기 변화 반복 | 없음 (고정 밝기) | 없음 (태양 아이콘으로 대체) |
| 별 반짝임 | 깜빡임 | 없음 (고정 밝기) | 없음 |
| 번개 | 화면 번쩍임 | 없음 (번개 아이콘만) | 없음 |

**채도(Saturation)**

| 프로필 | oklch chroma 조정 |
|--------|-------------------|
| Standard | 원본 유지 |
| Calm | chroma × 0.7 |
| Still | chroma × 0.4 (거의 무채색에 가까운 부드러운 톤) |

예시 — 맑은 날 배경:
- Standard: `oklch(92% 0.12 85)` → 선명한 노란 하늘
- Calm: `oklch(92% 0.084 85)` → 부드러운 노란 하늘
- Still: `oklch(92% 0.048 85)` → 거의 베이지에 가까운 따뜻한 톤

**소리**

| 프로필 | 소리 |
|--------|------|
| Standard | 선택적 앰비언트 사운드 (비소리, 바람소리 등). 기본 OFF, 사용자가 켤 수 있음 |
| Calm | 소리 완전 비활성. 토글 숨김 |
| Still | 소리 완전 비활성. 토글 숨김 |

> **원칙**: 소리는 절대 자동 재생하지 않는다. Standard 모드에서도 기본값은 무음이다. 앰비언트 사운드를 원하는 사용자만 명시적으로 켠다.

**모션 (prefers-reduced-motion 연동)**

시스템의 `prefers-reduced-motion: reduce` 설정이 감지되면 자동으로 Still 프로필을 기본값으로 적용한다. 보호자가 별도로 변경 가능.

#### 감각 모드 전환 UI

화면 우측 상단에 감각 모드 아이콘을 배치한다. 탭하면 3단계를 순환한다.

```
[☀️]  Standard → [🌙] Calm → [⏸️] Still → [☀️] Standard ...
```

아이콘은 현재 모드를 나타내며, 전환 시 화면 하단에 1.5초간 토스트로 모드명을 표시한다.

### 학습 단계 (Level 1 / 2 / 3)

나이와 인지 발달 수준에 따라 정보 복잡도를 조절한다. 보호자가 설정하며, 사용자의 성장에 따라 변경 가능.

#### Level 1: 한눈에 (5-7세)

**목표**: 글자를 모르거나 읽기 어려운 아이가 캐릭터 이미지와 이모지 하나로 날씨를 이해한다.

| 요소 | 표시 | 예시 |
|------|------|------|
| 캐릭터 | 전신, 날씨 맞는 옷 | (이미지) |
| 날씨 아이콘 | 대형 이모지 1개 | ☀️ / 🌧 / ❄️ |
| 텍스트 | 단어 1개 (큰 글씨) | "추워!" / "더워!" / "비!" |
| 온도 | 숨김 | — |
| 시간대별 | 숨김 | — |
| 옷차림 조언 | 캐릭터가 입고 있는 옷 자체가 조언 | — |

**시각 스케줄 통합** (Level 1 전용):

"지금 날씨 → 뭘 입을까 → 나가자!" 3단계를 화면 하단에 아이콘 시퀀스로 표시.

```
  [🌤️]  →  [🧥]  →  [🚪]
  날씨      옷       출발!
```

각 단계를 탭하면 해당 정보가 크게 표시된다. 현재 활성 단계에 밑줄 또는 하이라이트.

#### Level 2: 알아보기 (8-12세)

**목표**: 날씨 정보를 읽기 시작하고, 옷차림 조언을 문장으로 이해한다.

| 요소 | 표시 | 예시 |
|------|------|------|
| 캐릭터 | 전신, 날씨 맞는 옷 | (이미지) |
| 날씨 아이콘 | 이모지 + 한글 라벨 | ☀️ 맑음 / 🌧 비 |
| 텍스트 | 짧은 문장 | "자켓 입고 나가!" |
| 온도 | 큰 숫자 1개 | 15° |
| 시간대별 | 4칸 (오전/오후/저녁/밤) — 아이콘만 | ☀️ → ⛅ → 🌧 → 🌙 |
| 옷차림 조언 | 말풍선 스타일 텍스트 | "오후에 비 올 수 있어. 우산!" |

#### Level 3: 날씨 읽기 (13세+)

**목표**: 실제 날씨 데이터를 읽는 법을 배운다. 기온, 체감온도, 습도, 풍속 등.

| 요소 | 표시 | 예시 |
|------|------|------|
| 캐릭터 | 상반신 (공간 절약) | (이미지) |
| 날씨 상태 | 아이콘 + 텍스트 | ☀️ 맑음 |
| 온도 | 현재 + 체감 | 15° (체감 12°) |
| 시간대별 | 6시간 단위, 온도 숫자 포함 | 오전 12° → 오후 15° → 저녁 10° |
| 상세 정보 | 습도, 풍속, UV 지수 | 💧 65% · 💨 12km/h · ☀️ UV 3 |
| 옷차림 조언 | 이유 포함 문장 | "바람이 강해서 바람막이를 추천해요" |
| 장소 검색 | 활성화 | 검색 바 표시 |

#### 레벨 간 전환 원칙

- 보호자가 설정 화면에서 변경한다. 잠금 옵션 있음 (아이가 임의 변경 방지).
- 레벨을 올릴 때 "새로운 정보가 추가됐어요!" 안내를 1회 표시한다.
- 레벨을 낮출 때 정보가 줄어드는 것에 대한 안내 없음 (혼란 방지).

### 와이어프레임 — 기본 뷰

Smart Display 가로 모드 (1024 x 600 기준, Echo Show 8 / Nest Hub)

기존 디자인(Scene 70% + Info Panel 30%)에서 ASD 적응:
- 레이아웃 비율 유지하되 정보 패널을 단순화
- 항상 같은 위치에 같은 요소가 있도록 고정 레이아웃 적용

```
+-----------------------------------------------------------------------+
|  [감각모드 🌙]                              시애틀  ·  오후 3:00  [⚙️] |
|                                                                         |
|  +--------------------------------------------+  +------------------+  |
|  |                                            |  |                  |  |
|  |              ☁️    ☁️                      |  |    🌧  비        |  |
|  |                 🌧️🌧️                      |  |                  |  |
|  |                                            |  |    15°           |  |
|  |                                            |  |                  |  |
|  |          [캐릭터: 우산 쓴 모습]            |  |  +-----------+   |  |
|  |                                            |  |  | "자켓 입고 |   |  |
|  |                                            |  |  | 우산 챙겨!"|   |  |
|  |    ~~~~~ 물웅덩이 ~~~~~                   |  |  +-----------+   |  |
|  |                                            |  |                  |  |
|  +--------------------------------------------+  +------------------+  |
|                                                                         |
|  [🌤️ 날씨] ──── [🧥 옷] ──── [🚪 출발!]       ← 시각 스케줄 (L1만)   |
+-----------------------------------------------------------------------+
         장면 영역 (70%)                 정보 패널 (30%)
```

**고정 레이아웃 규칙**:
- 장면 영역: 항상 좌측 70%. 날씨가 바뀌어도 위치/크기 불변.
- 정보 패널: 항상 우측 30%. 구성 요소의 수직 순서 고정.
- 상단 바: 항상 동일 위치에 감각 모드 토글 + 위치명 + 시간 + 설정.
- 하단 시각 스케줄: Level 1에서만 표시. Level 2/3에서는 시간대별 날씨로 대체.

**정보 패널 수직 순서 (항상 고정)**:
1. 날씨 아이콘 + 상태 텍스트
2. 온도
3. 옷차림 조언 (말풍선)
4. (Level 2+) 시간대별 날씨
5. (Level 3) 상세 정보

### 와이어프레임 — 일과 통합 뷰

Level 1 전용. "오늘 하루 날씨 → 옷 입기 → 외출" 시각 스케줄을 메인 화면에 통합한다.

시각 스케줄을 탭하면 해당 단계가 장면 영역에 크게 표시된다.

**1단계: 날씨 (기본 화면)**

```
+-----------------------------------------------------------------------+
|  [🌙]                                       시애틀  ·  오전 8:00  [⚙️] |
|                                                                         |
|  +--------------------------------------------+  +------------------+  |
|  |                                            |  |                  |  |
|  |                                            |  |     ❄️           |  |
|  |            (눈 오는 장면)                  |  |                  |  |
|  |                                            |  |    "추워!"       |  |
|  |          [캐릭터: 패딩 입은 모습]          |  |                  |  |
|  |                                            |  |                  |  |
|  |                                            |  |                  |  |
|  +--------------------------------------------+  +------------------+  |
|                                                                         |
|  [🌤️ 날씨]━━━━[🧥 옷]────[🚪 출발!]                                   |
|   ▲ 활성                                                                |
+-----------------------------------------------------------------------+
```

**2단계: 옷 입기 (시각 스케줄 "옷" 탭)**

장면 영역에 캐릭터가 입어야 할 옷 아이템을 하나씩 보여준다.
아이가 "다음" 버튼으로 순서를 넘긴다. 체크리스트 형태.

```
+-----------------------------------------------------------------------+
|  [🌙]                                       시애틀  ·  오전 8:00  [⚙️] |
|                                                                         |
|  +--------------------------------------------+  +------------------+  |
|  |                                            |  |                  |  |
|  |                                            |  |   오늘 입을 옷   |  |
|  |          [캐릭터: 패딩 입은 모습]          |  |                  |  |
|  |                                            |  |   ✅ 긴팔       |  |
|  |             옷 아이템이                     |  |   ✅ 바지       |  |
|  |             하이라이트됨                    |  |   ⬜ 패딩       |  |
|  |             (현재: 패딩)                    |  |   ⬜ 목도리     |  |
|  |                                            |  |   ⬜ 장갑       |  |
|  +--------------------------------------------+  +------------------+  |
|                                                                         |
|  [🌤️ 날씨]────[🧥 옷]━━━━[🚪 출발!]                                   |
|               ▲ 활성                                                    |
+-----------------------------------------------------------------------+
```

**3단계: 출발! (시각 스케줄 "출발" 탭)**

캐릭터가 완전히 옷을 입고 준비된 모습. 격려 메시지.

```
+-----------------------------------------------------------------------+
|  [🌙]                                       시애틀  ·  오전 8:00  [⚙️] |
|                                                                         |
|  +--------------------------------------------+  +------------------+  |
|  |                                            |  |                  |  |
|  |                                            |  |                  |  |
|  |          [캐릭터: 완전 무장 모습]          |  |   "준비 완료!"   |  |
|  |           밝은 표정, 엄지척                |  |                  |  |
|  |                                            |  |   👍            |  |
|  |                                            |  |                  |  |
|  |                                            |  |   "좋은 하루     |  |
|  |                                            |  |    보내!"        |  |
|  +--------------------------------------------+  +------------------+  |
|                                                                         |
|  [🌤️ 날씨]────[🧥 옷]────[🚪 출발!]━━━━                               |
|                           ▲ 활성                                        |
+-----------------------------------------------------------------------+
```

**일과 통합 원칙**:
- 단계는 항상 왼쪽→오른쪽 순서. 건너뛰기 가능하지만 순서는 바뀌지 않는다.
- 활성 단계는 굵은 선(━)과 밑줄로 표시. 비활성은 얇은 선(─).
- 각 단계 아이콘의 위치가 절대 바뀌지 않는다.
- 시각 스케줄 바는 화면 하단에 고정. 스크롤 영향 없음.

### 전환/애니메이션 가이드

#### 날씨 전환 (장소 변경 또는 시간 경과로 날씨가 바뀔 때)

**문제**: 갑작스러운 화면 전환은 ASD 사용자에게 불안을 유발할 수 있다.

**해결 — 전환 예고 시스템**:

1. **전환 전 예고 (2초)**: 정보 패널에 "날씨가 바뀌고 있어요" 텍스트 + 로딩 표시가 먼저 나타난다.
2. **장면 전환**: 이전 장면이 서서히 사라지고 새 장면이 나타난다 (크로스페이드).
3. **전환 후 안정 (1초)**: 모든 요소가 자리 잡힌 후 정보 패널 텍스트가 업데이트된다.

| 프로필 | 예고 | 전환 시간 | 방식 |
|--------|------|-----------|------|
| Standard | "날씨가 바뀌어요" 1초 | 600ms 크로스페이드 | 배경 + 캐릭터 동시 전환 |
| Calm | "날씨가 바뀌어요" 2초 | 1200ms 크로스페이드 | 배경 먼저 → 캐릭터 후 (순차) |
| Still | "날씨가 바뀌어요" 2초 | 0ms (즉시) | 즉시 전환하되 예고로 예측 가능성 확보 |

#### 일과 단계 전환 (시각 스케줄 탭 이동)

- 전환 방식: 장면 영역만 좌→우 슬라이드 (정보 패널 고정)
- 속도: Standard 300ms / Calm 600ms / Still 0ms
- 정보 패널은 절대 움직이지 않는다 (앵커 역할)

#### 캐릭터 이미지 교체

- 같은 위치, 같은 크기로 교체한다. 위치 이동 없음.
- 페이드 전환: 이전 이미지 opacity 1→0, 새 이미지 opacity 0→1
- Standard: 300ms / Calm: 600ms / Still: 0ms

#### 터치 피드백

- 버튼 탭: `transform: scale(0.96)` + 100ms (Standard/Calm), 없음 (Still)
- 시각 스케줄 탭: 탭한 단계에 배경색 하이라이트 (200ms)
- 긴 누르기(long press): 사용하지 않음
- 더블 탭: 사용하지 않음
- 스와이프: 사용하지 않음 (모든 인터랙션은 단일 탭)

#### 금지 애니메이션 (ASD 환경 전체)

어떤 프로필에서든 사용하지 않는 것:
- 화면 흔들림(shake)
- 깜빡임(blink, flash)
- 자동 스크롤/캐러셀
- 팝업/모달 (화면 위 레이어)
- 갑작스러운 소리 재생

### 디자인 시스템 수정사항

기존 02-design.md의 디자인 시스템을 ASD 모드에서 오버라이드하는 토큰들.

#### 추가 CSS 변수

```css
/* 감각 프로필 토큰 */
--sensory-chroma-scale: 1;        /* Standard: 1, Calm: 0.7, Still: 0.4 */
--sensory-transition-speed: 1;    /* Standard: 1, Calm: 0.5, Still: 0 */
--sensory-particle-opacity: 1;    /* Standard: 1, Calm: 0, Still: 0 */

/* 전환 속도 계산 */
--transition-scene: calc(600ms * var(--sensory-transition-speed));
--transition-character: calc(300ms * var(--sensory-transition-speed));
--transition-slide: calc(300ms * var(--sensory-transition-speed));

/* 레벨별 타이포그래피 */
--text-weather-label-l1: 2rem / 1.2;      /* Level 1: 단어 1개를 크게 */
--text-weather-label-l2: 1.5rem / 1.3;    /* Level 2: 기존 --text-advice */
--text-weather-label-l3: 1.125rem / 1.4;  /* Level 3: 기존 --text-temp */

/* 레벨별 아이콘 크기 */
--icon-weather-l1: 4rem;    /* Level 1: 대형 이모지 */
--icon-weather-l2: 2.5rem;  /* Level 2: 중형 */
--icon-weather-l3: 1.5rem;  /* Level 3: 소형, 인라인 */

/* 시각 스케줄 */
--schedule-height: 3.5rem;
--schedule-icon-size: 2rem;
--schedule-active-color: oklch(65% 0.15 250);
--schedule-inactive-color: oklch(75% 0.02 0);
```

#### 배경색 오버라이드

기존 `--bg-*` 변수에 chroma 스케일을 적용:

```css
/* 예: Calm 모드 비 배경 */
/* 원본: oklch(70% 0.04 250) */
/* Calm:  oklch(70% calc(0.04 * 0.7) 250) = oklch(70% 0.028 250) */
/* Still: oklch(70% calc(0.04 * 0.4) 250) = oklch(70% 0.016 250) */
```

> 실제 구현에서는 JS로 oklch chroma 값을 동적 계산하여 CSS 변수에 주입한다.

#### 터치 타겟 확장

ASD 사용자의 미세 운동 기술 어려움을 고려하여 기존 48px 최소 기준을 확장:

| 요소 | 기본 디자인 | ASD 모드 |
|------|------------|----------|
| 일반 버튼 | 48px | 56px |
| 시각 스케줄 탭 | — | 64px (넓은 탭 영역) |
| 감각 모드 토글 | — | 48px |
| 설정 아이콘 | 48px | 48px (3초 롱프레스 접근) |
| 체크리스트 항목 | — | 56px 높이, 전체 행 탭 가능 |

#### 포커스 스타일

키보드/스위치 접근성을 위한 포커스 링:

```css
/* 기본 디자인의 포커스 링 유지 */
:focus-visible {
  outline: 3px solid oklch(65% 0.15 250);
  outline-offset: 2px;
}

/* ASD 모드: 포커스 링을 더 두껍게 */
[data-sensory="calm"] :focus-visible,
[data-sensory="still"] :focus-visible {
  outline-width: 4px;
  outline-offset: 3px;
}
```

#### 탭 순서 (키보드/스위치 내비게이션)

```
1. 감각 모드 토글
2. 위치명 (읽기 전용, aria-label로 위치 안내)
3. 설정 아이콘
4. 정보 패널 — 날씨 상태 (읽기 전용)
5. 정보 패널 — 옷차림 조언 (읽기 전용)
6. (Level 1) 시각 스케줄 — 날씨 탭
7. (Level 1) 시각 스케줄 — 옷 탭
8. (Level 1) 시각 스케줄 — 출발 탭
9. (Level 2+) 시간대별 날씨 영역
10. (Level 3) 검색 입력
```

#### 캐릭터를 소셜 스토리 도구로 활용

소셜 스토리(Social Story)는 ASD 아동에게 사회적 상황을 설명하는 교육 기법이다. 캐릭터가 날씨 상황에 대응하는 모습 자체가 소셜 스토리 역할을 한다.

**Level 1 소셜 스토리 시퀀스**:

| 단계 | 캐릭터 모습 | 의미 |
|------|------------|------|
| 1. 날씨 | 캐릭터가 창문 밖을 보는 모습 | "밖을 보니 비가 와요" |
| 2. 옷 | 캐릭터가 옷을 입는 모습 | "비가 오니까 이 옷을 입어요" |
| 3. 출발 | 캐릭터가 준비된 모습으로 웃음 | "준비 완료! 나갈 수 있어요" |

이 시퀀스는:
- 항상 같은 3단계 순서로 진행된다 (예측 가능)
- 캐릭터의 표정이 긍정적이다 (불안하거나 슬프지 않음)
- 각 단계에서 한 가지만 설명한다 (인지 부하 최소)
- 아이가 자기 속도로 넘길 수 있다 (자동 진행 없음)
